## Actionable Issues

### Issue 1: BET-02/03/04 defaults wrong in settings.py (HIGH)

`services/stake/settings.py` SizingSettings defaults:
- `kelly_multiplier=0.5` should be `0.25` (BET-02)
- `per_bet_cap_pct=0.05` should be `0.03` (BET-03)
- `max_total_exposure_pct=0.10` should be `0.05` (BET-04)
- `max_win_bets=3` should be `2` (BET-04)

The math functions in `math.py` have correct defaults, but `sizing_node` always passes settings values, so runtime behavior uses the wrong (more aggressive) defaults.

### Issue 2: drawdown_unlock_kb() never attached (MEDIUM)

`drawdown_unlock_kb()` is defined in `keyboards/stake_kb.py`, imported in `handlers/results.py`, but never called. When drawdown fires, user sees skip text with no button. Fix: attach `reply_markup=drawdown_unlock_kb()` in `_run_analysis_inline` when `skip_tier == 0`.

### Issue 3: Phase 1 & 2 lack VERIFICATION.md (TECH DEBT)

These phases were executed before the verifier agent was standard. Code is tested (210 tests pass) and integration checker confirms all wiring works. Missing formal verification is tech debt, not a blocker.
