    ```yaml
          - 8.8.8.8

          - STAKE_DATABASE_PATH=/app/data/races.db
          - STAKE_AUDIT__LOG_PATH=/app/data/stake_audit.jsonl
          - STAKE_TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
          - STAKE_TELEGRAM_CHAT_ID=${TELEGRAM_CHAT_ID}
          - STAKE_OPENROUTER_API_KEY=${OPENROUTER_API_KEY}
        volumes:
          - db_data:/app/data
        restart: unless-stopped
        networks:
          - racehorse-network
    ```

    IMPORTANT: The `STAKE_` prefix env vars map to `StakeSettings` fields. The `${TELEGRAM_BOT_TOKEN}` etc. reference existing .env vars so no new secrets are needed.

    **Step 3: Update entrypoint.sh**
    Add stake migrations call. Update the python3 -c block to also run stake migrations:

    ```bash
    #!/bin/bash
    # Run database migrations before starting service
    python3 -c "
    from src.database.migrations import run_migrations, initialize_default_agents
    import os
    db_path = os.environ.get('RACEHORSE_DATABASE__PATH', os.environ.get('STAKE_DATABASE_PATH', 'races.db'))
    run_migrations(db_path)
    initialize_default_agents(db_path)
    # Stake-specific migrations
    try:
        from services.stake.bankroll.migrations import run_stake_migrations
        run_stake_migrations(db_path)
    except ImportError:
        pass  # Not all services need stake migrations
    "

    # Start the service
    exec "$@"
    ```

    The `try/except ImportError` ensures existing services (monitor, orchestrator, etc.) are not broken by the new import.
